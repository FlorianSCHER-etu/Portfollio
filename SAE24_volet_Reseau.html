Architecture réseau complète pour Fibre&Company - Focus sur l'infrastructure LAN

Présentation du projet

Contexte professionnel

Employé chez Fibre&Company, société spécialisée dans le déploiement de la fibre optique, j'ai été chargé de concevoir et implémenter l'infrastructure réseau LAN de l'entreprise.

Ma mission - Infrastructure LAN

  • Configuration et gestion des VLANs selon l'organigramme
  • Implémentation du routage inter-VLAN
  • Mise en place de liens agrégés entre commutateurs
  • Configuration NAT et ACL pour la sécurité
  • Services DHCP et spanning tree
Durée : 3 semaines
Travail en équipe

Mes réalisations techniques

Configuration des VLANs

Mise en place de la segmentation réseau selon l'organigramme de l'entreprise

Réalisations :
  • Création de 6 VLANs métier + VLAN 777 (sécurité)
  • Attribution des ports selon les besoins
  • Configuration du VLAN natif pour la sécurité
  • Calcul optimal des masques de sous-réseau

Routage Inter-VLAN

Configuration du routage L3 sur le switch Cisco 3750 pour interconnecter les VLANs

Réalisations :
  • Activation du routage IP sur le switch 3750
  • Configuration des interfaces VLAN (SVI)
  • Règles de routage : accès serveurs depuis tous les VLANs
  • Isolement inter-VLAN pour la sécurité

Liens agrégés et Spanning Tree

Mise en place de la redondance et optimisation des performances

Réalisations :
  • Configuration EtherChannel entre les 3 switches
  • Liens agrégés 2x1Gb entre chaque switch
  • Configuration RSTP avec le 3750 comme root bridge
  • Optimisation des chemins de redondance

Services DHCP

Configuration des pools DHCP pour l'attribution automatique des adresses IP

Réalisations :
  • Configuration DHCP sur le switch 3750
  • Pools DHCP pour chaque VLAN utilisateur
  • Exclusion du VLAN serveurs (IP fixes)
  • DNS par défaut : 1.1.1.1 (Cloudflare)

Sécurisation et ACL

Implémentation des mesures de sécurité réseau et contrôle d'accès

Réalisations :
  • ACL pour bloquer le trafic inter-VLAN non autorisé
  • Configuration du VLAN 777 pour ports non utilisés
  • Désactivation Telnet/HTTP, activation SSH
  • Sécurisation des ports d'accès (PortSecurity)

Défis techniques relevés

Calcul des sous-réseaux

Optimisation du plan d'adressage avec calcul précis des masques selon les besoins réels de chaque service.

Solution : Utilisation du VLSM pour minimiser le gaspillage d'adresses IP

Isolation des VLANs

Garantir l'accès aux serveurs depuis tous les VLANs tout en bloquant la communication inter-VLAN.

Solution : ACL granulaires permettant seulement le trafic vers VLAN serveurs

Équilibrage de charge

Configuration optimale des liens agrégés pour la répartition du trafic et la redondance.

Solution : EtherChannel avec LACP pour la répartition automatique
