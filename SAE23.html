Application web météorologique interactive et responsive avec carte Leaflet

Durée : 6 semaines Travail : Individuel Technologies : HTML, CSS, JavaScript BUT RT - Projet individuel

Contexte du projet

La SAE 2.03 "Instant Weather V2" consiste à développer une application web météorologique complète utilisant plusieurs APIs externes. L'objectif est de créer une interface moderne, responsive et accessible permettant d'afficher les prévisions météo de n'importe quelle commune française.

Objectifs principaux

  • Utiliser une API conformément à sa documentation (Météo Concept)
  • Manipuler le DOM pour intégrer dynamiquement du contenu
  • Interagir avec la mise en forme via JavaScript
  • Créer une interface utilisateur moderne et intuitive
  • Implémenter des fonctionnalités avancées (carte, historique, thèmes)
  • Assurer la compatibilité mobile et l'accessibilité

Évolution depuis la V1

  • Sélection du nombre de jours de prévision (1 à 7 jours)
  • Informations optionnelles (latitude/longitude, précipitations, vent)
  • Carte interactive avec localisation
  • Historique des dernières recherches
  • Mode sombre/clair dynamique
  • Interface responsive et accessible

Ce que je retiens

Ce projet m'a permis d'approfondir considérablement mes compétences en JavaScript moderne et en intégration d'APIs. La gestion de plusieurs sources de données externes m'a fait découvrir l'importance de l'architecture logicielle et de la gestion d'erreurs robuste.

L'implémentation de fonctionnalités avancées comme la cartographie interactive et le système de thèmes m'a montré la richesse de l'écosystème JavaScript. La contrainte d'accessibilité m'a sensibilisé à l'importance de créer des applications véritablement inclusives.

Cette expérience m'a également appris l'importance de la documentation et du versioning propre, essentiels pour la maintenance et l'évolution d'un projet. L'approche mobile-first s'est révélée crucial pour une expérience utilisateur optimale.

Difficultés rencontrées

Intégration multi-APIs

La coordination entre l'API Géo et Météo Concept s'est révélée complexe, notamment pour la gestion des erreurs en cascade et la synchronisation des données. Les formats de réponse différents ont nécessité des adaptateurs spécifiques.

Intégration de Leaflet

L'adaptation de la carte aux thèmes sombre/clair et la gestion responsive des marqueurs ont demandé une compréhension approfondie de l'API Leaflet. La synchronisation entre les données météo et la carte a nécessité plusieurs itérations.

Problème technique GitHub

L'inaccessibilité temporaire du compte GitHub principal a créé une rupture dans le workflow habituel. La création d'un compte secondaire pour le déploiement a permis de contourner le problème tout en maintenant la continuité du projet.

Axes d'amélioration

Fonctionnalités avancées

Intégration de graphiques interactifs pour visualiser l'évolution des températures et précipitations. Ajout de notifications push pour les alertes météo importantes.

Plan d'action : Formation aux bibliothèques de visualisation (Chart.js, D3.js) et étude des APIs de notification web.

Persistance des données

Migration vers IndexedDB pour un stockage local plus robuste et implémenter un cache intelligent des données météo pour réduire les appels API.

Plan d'action : Étude d'IndexedDB et des service workers pour la mise en cache avancée et le fonctionnement hors ligne.

Tests automatisés

Mise en place d'une suite de tests unitaires et d'intégration pour garantir la fiabilité de l'application, notamment pour les interactions avec les APIs externes.

Plan d'action : Formation aux frameworks de test JavaScript (Jest, Cypress) et intégration dans le workflow CI/CD.
