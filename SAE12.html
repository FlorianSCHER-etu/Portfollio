SAE 1.02 – Administration des systèmes et réseaux

Configuration et administration de services réseau sur machine virtuelle

Durée : 6 semaines Travail : Groupe de 5 étudiants Services : DHCP, DNS, HTTP, SSH BUT RT - Administration réseau

Contexte du projet

La SAE 1.02 vise à développer les compétences en administration de systèmes et réseaux. L'objectif est de configurer et administrer des services réseau sur une machine virtuelle pour simuler un environnement professionnel et comprendre les enjeux de l'infrastructure IT.

Objectifs principaux

  • Déployer et configurer une machine virtuelle Linux
  • Installer et configurer des services réseau essentiels
  • Sécuriser les services avec pare-feu et bonnes pratiques
  • Documenter toutes les configurations pour reproductibilité
  • Tester le bon fonctionnement des services déployés
  • Travailler en équipe sur un projet d'infrastructure

Services réseau déployés

  • Serveur DHCP pour attribution automatique d'adresses IP
  • Serveur DNS pour résolution de noms de domaine
  • Serveur HTTP/HTTPS avec Apache ou Nginx
  • Serveur SSH pour administration à distance
  • Serveur de fichiers avec Samba ou NFS
  • Configuration de pare-feu avec UFW/iptables

Infrastructure mise en place

Machine virtuelle Ubuntu Server

  • OS : Ubuntu Server 20.04 LTS
  • Virtualisation : VirtualBox avec configuration réseau adaptée
  • Ressources : 2 CPU, 4GB RAM, 20GB stockage
  • Réseau : Configuration IP fixe et accès Internet
  • Accès distant : SSH configuré pour administration

Services réseau configurés

  • DHCP Server : Attribution automatique d'IPs dans un réseau local simulé
  • DNS Server : Résolution locale avec zones personnalisées
  • Web Server : Apache/Nginx avec page d'accueil personnalisée
  • File Server : Partage de fichiers via Samba
  • Security : Pare-feu UFW avec règles restrictives

Sécurisation mise en œuvre

  • Pare-feu : UFW configuré avec ouverture sélective des ports
  • SSL/TLS : Certificats pour sécuriser le serveur web
  • SSH : Authentification par clés et désactivation root
  • Monitoring : Surveillance des services et logs système

Sécurisation mise en place

  • Politique par défaut : tout refuser en entrée

  • Limitation du taux de connexion SSH
  • Logs de connexions pour audit

Bonnes pratiques sécurité

  • Désactivation du compte root pour SSH
  • Authentification SSH par clés uniquement
  • Mise à jour automatique des paquets sécurité
  • Surveillance des tentatives d'intrusion

Ce que je retiens

Ce projet m'a permis de découvrir concrètement l'administration système et réseau en environnement Linux. La mise en place de services réseau essentiels m'a fait comprendre l'importance de l'infrastructure dans le fonctionnement d'un système d'information moderne.

L'aspect le plus enrichissant fut la compréhension des interactions entre les différents services : comment le DHCP, DNS et les services applicatifs s'articulent pour créer un environnement fonctionnel. Le travail en équipe m'a aussi appris l'importance de la documentation et de la coordination sur des projets techniques.

La dimension sécurité, souvent négligée au début, s'est révélée cruciale pour un déploiement professionnel. Chaque service doit être sécurisé individuellement tout en maintenant la cohérence globale du système.

Difficultés rencontrées

Configuration réseau complexe

La gestion des différents modes réseau de VirtualBox (NAT, bridge, host-only) et leur impact sur la connectivité des services s'est révélée plus délicate que prévu. Il a fallu comprendre les subtilités de chaque mode pour choisir la configuration optimale.

Interdépendances des services

Les services réseau sont fortement interdépendants : un problème de DNS peut affecter le DHCP, qui lui-même impacte l'accès web. Il a fallu apprendre à diagnostiquer et résoudre ces problèmes en chaîne, souvent en analysant les logs système.

Coordination d'équipe technique

Travailler à 5 sur une infrastructure partagée nécessite une coordination rigoureuse pour éviter les conflits de configuration. La mise en place d'une méthodologie de travail avec Git et la documentation continue ont été des défis importants.

Axes d'amélioration

Automatisation avancée

Développer des scripts d'automatisation plus sophistiqués avec Ansible ou des outils d'Infrastructure as Code pour rendre le déploiement encore plus reproductible et efficace.

Plan d'action : Formation aux outils DevOps et développement de playbooks Ansible pour automatiser les déploiements.

Monitoring et supervision

Intégrer des outils de monitoring plus avancés (Nagios, Prometheus, Grafana) pour surveiller proactivement les performances et la disponibilité des services déployés.

Sécurité renforcée

Approfondir les aspects sécurité avec audit de sécurité automatisé, détection d'intrusion et mise en place de politiques de sécurité plus strictes.
