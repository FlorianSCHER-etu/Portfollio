SAE 1.04 – Se présenter sur Internet

Développement d'un site web responsive sur le thème du handball

Durée : 4 semaines Travail : Individuel Site : Handball AirForce BUT RT - Développement web

Contexte du projet

La SAE 1.04 vise à développer les compétences en création de sites web statiques. L'objectif est de créer un site web complet, responsive et conforme aux standards du web, en utilisant uniquement HTML et CSS.

Objectifs principaux

  • Création d'un site web de minimum 3 pages
  • Mise en place d'un design responsive (mobile, tablette, desktop)
  • Respect des standards W3C et de l'accessibilité WCAG 2.0 AA
  • Utilisation de Flexbox et/ou CSS Grid
  • Intégration d'animations CSS
  • Hébergement sur GitHub Pages

Thème choisi - Handball

  • Présentation du sport et de ses règles
  • Histoire et évolution du handball
  • Équipes et compétitions importantes
  • Techniques et tactiques de jeu
  • Matériel et équipement

Architecture du site réalisé

Page d'accueil (index.html)

  • Header avec navigation : Menu responsive avec liens vers toutes les pages
  • Section hero : Image d'accroche avec animation CSS au chargement
  • Présentation générale : Introduction au handball avec visuels
  • Actualités : Section avec les dernières news du handball
  • Footer : Informations de contact et liens sociaux

Page dédiée au sujet (handball.html)

  • Histoire du handball : Origines et développement du sport
  • Règles du jeu : Explication détaillée des règles principales
  • Positions et rôles : Description des différents postes
  • Équipement : Matériel nécessaire pour jouer
  • Compétitions : Présentation des championnats majeurs

Page à propos (about.html)

  • Présentation personnelle : Parcours et formation
  • Motivation : Pourquoi avoir choisi le handball
  • Compétences techniques : Technologies maîtrisées
  • Contact : Moyens de communication

Pages supplémentaires

  • Galerie (gallery.html) : Photos et vidéos de handball
  • Contact (contact.html) : Formulaire de contact stylisé

Technologies et techniques utilisées

Technologies de base

HTML5

  • Sémantique : Utilisation appropriée des balises sémantiques (header, nav, main, section, article, aside, footer)
  • Accessibilité : Attributs alt pour les images, labels pour les formulaires
  • Métadonnées : Balises meta pour le responsive et le SEO
  • Validation W3C : Code HTML validé sans erreur

CSS3

  • Flexbox : Mise en page flexible pour la navigation et les sections
  • CSS Grid : Grille pour l'organisation des galeries d'images
  • Media Queries : Breakpoints pour mobile (768px) et tablette (1024px)
  • Animations : Transitions au survol et animations keyframes
  • Variables CSS : Utilisation de custom properties pour la cohérence

Fonctionnalités avancées

Design Responsive

  • Mobile First : Approche mobile d'abord puis adaptation desktop
  • Navigation adaptative : Menu hamburger sur mobile
  • Images responsives : Redimensionnement automatique des visuels
  • Typographie fluide : Tailles de police adaptées aux écrans

Animations CSS

  • Animation d'entrée : Apparition progressive du contenu au chargement
  • Effets de survol : Transitions sur les boutons et liens
  • Animation des images : Zoom et effets sur les photos de galerie
  • Navigation animée : Transition du menu mobile

Accessibilité WCAG 2.0 AA

  • Contraste : Ratio de contraste respecté pour le texte
  • Navigation clavier : Tous les éléments accessibles au clavier
  • Textes alternatifs : Descriptions des images pour les lecteurs d'écran
  • Structure logique : Hiérarchie des titres cohérente

Techniques mises en œuvre

Responsive Design

Breakpoints utilisés

  • Mobile : < 768px - Navigation hamburger, layout vertical
  • Tablette : 768px - 1024px - Adaptation intermédiaire
  • Desktop : > 1024px - Layout complet horizontal

Techniques responsive

  • Unités relatives (rem, em, %, vh/vw)
  • Images flexibles avec max-width: 100%
  • Flexbox pour l'alignement et la distribution
  • CSS Grid pour les mises en page complexes

Animations CSS réalisées

Animation d'entrée (page d'accueil)

  • Fade-in progressif du contenu au chargement
  • Translation verticale avec effet d'apparition
  • Délais échelonnés pour un effet de cascade

Interactions utilisateur

  • Hover effects sur les boutons avec changement de couleur
  • Zoom au survol des images de galerie
  • Transition du menu mobile avec transformation
  • Animation des liens de navigation

Validation et conformité

Validation W3C

  • HTML : 0 erreur, 0 avertissement
  • CSS : 0 erreur, validation complète

Ce que je retiens

Ce projet m'a permis de découvrir concrètement le développement web front-end et de comprendre l'importance des standards du web. La création d'un site entièrement responsive m'a fait réaliser la complexité de l'adaptation multi-supports et l'importance de penser mobile-first.

L'aspect le plus enrichissant fut la maîtrise de Flexbox et CSS Grid, deux technologies puissantes qui révolutionnent la mise en page web. Le travail sur l'accessibilité m'a sensibilisé à l'inclusion numérique et aux bonnes pratiques pour rendre le web accessible à tous.

L'utilisation de Git et GitHub pour le versioning m'a initié aux bonnes pratiques de développement collaboratif. La possibilité d'héberger gratuitement sur GitHub Pages est un atout majeur pour présenter ses projets.

Difficultés rencontrées

Complexité du responsive design

La gestion des différents breakpoints et l'adaptation du contenu sur tous les supports s'est révélée plus complexe que prévu. Certains éléments nécessitaient des ajustements spécifiques pour chaque taille d'écran. J'ai rencontré beaucoup de problème avec les media query pour l'affichage clair sur mobile

Respect de l'accessibilité

Au moment de la validation des normes W3C, j'ai eu bcp d'erreur fatal que j'ai réussi à corriger. Pour ce faire j'ai du comprends les erreurs au lignes indiqués, j'avais oublié de fermé certaines balise, j'avais mis des balise strong avec des balise h2 dedans.

Compatibilité navigateurs

Quelques propriétés CSS récentes n'étaient pas supportées de manière uniforme. Il a fallu prévoir des solutions de fallback et tester sur plusieurs navigateurs pour garantir un rendu cohérent.

Axes d'amélioration

Optimisation des performances

Mettre en place des techniques d'optimisation avancées : compression d'images, minification CSS, lazy loading, et mesure des performances avec Lighthouse.

Plan d'action : Formation aux outils d'optimisation web et intégration d'un processus de build automatisé.

Intégration d'un CMS

Évoluer vers un site dynamique avec système de gestion de contenu pour faciliter les mises à jour et l'ajout de nouvelles fonctionnalités.
